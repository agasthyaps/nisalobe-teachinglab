## Teaching Lab CAP Knowledge Base

This knowledge base contains essential guidance for Teaching Lab's Coaching Action Plan (CAP) system. Reference this consistently when supporting coaches with CAP-related questions, troubleshooting, or process clarification.

### Navigate
- `capoverview.md`: Conceptual overview of CAP structure, purpose, and quality exemplars
- `CAPguide.md`: Complete reference guide for CAP creation, naming, storage, and authorization
- `updates.md`: Chronological updates including fixes, FAQs, and system changes

### Contents

#### **capoverview.md**
Foundational document explaining what CAPs are, how they should be used, and what quality entries look like.
- CAP definition and three coaching models (1:1, group, leader)
- Core structure: cycles, sessions, key components
- Detailed breakdown of Area of Focus, Goals, Metrics, Menu of Coaching Moves, Decision Log, and Session Documentation
- Step-by-step workflow for using CAPs throughout coaching cycles
- Exemplar entries with explanations of what makes them strong
- Common pitfalls and how to avoid them

**Key use cases:**
- Coach or staff member is new to CAPs and needs foundational understanding
- Coach asks "what should I write in [Area of Focus/Metrics/Session Notes]?"
- Coach needs examples of high-quality CAP entries
- Explaining the difference between coaching moves, sessions, and cycles
- Clarifying what makes metrics observable and measurable
- Teaching someone how to document coaching impact effectively

#### **CAPguide.md**
Core reference document covering CAP system fundamentals.
- Naming conventions for 1:1, group, and leader coaching models
- File storage organization and structure
- Authorization and setup procedures
- Data integrity requirements and common pitfalls
- Group coaching rostering process
- Handling coaching sessions beyond standard cycle limits

**Key use cases:**
- Coach asks how to name a new CAP
- Coach needs to troubleshoot authorization issues
- Coach is unclear about file storage location
- Coach has questions about group vs. individual CAP setup

#### **updates.md**
Bi-weekly "This Month in CAP-Land" newsletter archive with ongoing fixes, finds, and FAQs.
- Known issues and workarounds (grammar/punctuation in dropdown validation)
- Process updates (group coaching, rostering changes)
- Data quality guidance (test CAP cleanup, folder organization)
- Best practice examples with screenshots
- Common errors and how to fix them

**Key use cases:**
- Coach reports dropdown validation errors (punctuation issue)
- Coach asks about recent changes to group coaching process
- Coach needs examples of well-written CAP sections
- Coach asks about micro-PL logging or specific edge cases

### Usage Guidelines

When coaches ask CAP-related questions:
1. For conceptual or "what/how" questions: Start with `capoverview.md` to establish foundational understanding
2. For recent changes or known issues: Check `updates.md` for current clarifications and workarounds
3. For procedural/technical setup: Reference `CAPguide.md` for naming, storage, and authorization processes
4. Always cite specific sections when providing guidance

For new coaches or conceptual questions: Start with `capoverview.md` to explain structure, purpose, and quality standards.

For naming questions: Direct to the master survey input and validation tool referenced in `CAPguide.md`.

For "what should I write" questions: Use exemplars from `capoverview.md` to show strong entries and explain quality indicators.

For technical issues: Check `updates.md` for recent system updates or known bugs before troubleshooting.

For process questions: Confirm which coaching model (1:1, group, or leader) before citing relevant sections.
